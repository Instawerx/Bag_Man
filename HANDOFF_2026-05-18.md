## Gotchas (the small, painful, easy-to-forget ones)

1. **UE Editor open → .uasset files write-locked** — git pull/merge/checkout/stash fails on `Content/**/*.uasset` while UE has the project loaded. Always close UE before any working-tree-changing git op. (Memory: `feedback_ue_open_locks_uassets.md`.)
2. **Orchestrator commits queue.yaml into yolo PRs via `git add -A`** — every PR ships with `status: running` for the current task; a cleanup commit on main flips it to `merged` and drops `last_attempt_at`. Pattern we keep repeating; not yet fixed because `commit_all()` uses `add -A`. If a yolo PR's diff looks odd, this is why.
3. **PowerShell NativeCommandError on successful git** — `git push origin main 2>&1` from PowerShell triggers an exception even on a successful push because git writes "To https://..." to stderr. Look at `[main XXX]` + `Updating XXX..YYY` lines + `$LASTEXITCODE`, not the red text.
4. **Anthropic rate window** — as of 2026-05-18 evening, overage was `rejected` (out_of_credits). 5h window resets, but if you start a task with low time left in the window, mid-task rate-limit hangs can occur. Stream-json mode + the watchdog now make this safe to test — worst case, watchdog kills after 10 min.
5. **PowerShell vs WSL bash process namespaces** — `kill -0 <pid>` in a Monitor's bash command can't see native Windows PIDs. Use `Get-Process -Id <pid>` from PowerShell to probe process liveness reliably.
6. **AFL-0110 queue entry is stale** — the queue.yaml entry still describes greybox authoring. The new Option-A scope is in `memory/project_afl0110_redirect_shootergym.md`. When AFL-0110 comes up, edit queue.yaml first.
