◆ Known Debt — named, tracked, must clear before the gate it points at

Real findings surfaced during build. NOT "harmless warnings" — every entry here is a Phase-blocker for the gate listed. The class of problem we used to ignore.

BM-DEBT-001 B_Experience_BagMan (and BagMan content generally) lives in /Game/ and references /ShooterCore/ plugin assets Blocks: Phase 1 plugin authoring
Lyra's asset-reference-restriction validator flagged 4 illegal references on save: /Game/BagMan/Experiences/B_Experience_BagMan referencing /ShooterCore/Game/HeroData_ShooterGame and the three LAS_ShooterGame_* ActionSets. Did not block save, but violates Lyra's content-dependency architecture: project content (/Game/) should not depend on plugin content (/ShooterCore/) — instead, plugin content activates into the project.

Proper fix: BagMan content belongs in a BagMan GameFeature plugin that activates ShooterCore content into the project, not /Game/ content depending directly on plugin content. The current arrangement works for Sprint 0.0 isolation but is the wrong shape for adding BagMan-specific abilities, weapons, or experiences in Phase 1+.

Why it matters: this is the exact "just a warning" class that silently bit the prior build. Named, not trusted to evaporate.
Surfaced 2026-05-26 during BM-0002 PIE diagnosis (PIE log line 4092). Must resolve before authoring BagMan-specific plugin content in Phase 1.

⌗ Why This Tracker Exists (post-mortem, kept on purpose)

The failure: "Adopt the Lyra character and fully expand it" was a Phase-1 foundational mission. It was marked ✅ and built upon for weeks — through Pulse, beam, heat, dash, tuning — while the possessed pawn never actually accepted input. WASD/LMB were dead the entire time.

The root deviation: a raw-C++ hero component with no BP child left Lyra's DefaultInputMappings empty at runtime (confirmed in the debugger). Lyra's InitializePlayerInput clears all mappings then repopulates from that array — empty on a raw-C++ class — so every binding silently died. The fix for that one layer was correct but insufficient; deeper input-routing layers remained.

The meta-failure: ✅ measured "code authored + compiled," not "feature demonstrated." Five separate "tracker said shipped, content was incomplete" findings surfaced in a single session. The checkbox lied because nobody watched the pawn move.

The correction: foundation first, canonical adoption, an immovable PIE-control gate, ✅ = watched-working, one layer at a time. This document is built so that trap cannot recur.
